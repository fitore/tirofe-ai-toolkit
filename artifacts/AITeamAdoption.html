The Problem with Human-in-the-Loop

Agents can generate code faster than humans can review, comprehend, or trust it. Teams with high AI adoption complete 21% more tasks and merge 98% more PRs — but spend 91% more time on review. We accelerated code generation without equally accelerating verification and trust. The pull request, designed for human-speed authorship, becomes a traffic jam at agent speed.

The current vision compounds this by placing humans and agents on the same execution path. Every stage requires a human gate before the agent can proceed. The system moves at human speed, negating most throughput gains. The fix is not to remove human oversight — it is to decouple the loops. Humans move upstream into constraints and intent; agents execute within them continuously.

✗ Current: Synchronous Gates
  • ✗Human approves each unit of agent work
  • ✗Agent blocked until human reviews
  • ✗System speed = human speed
  • ✗Human becomes the bottleneck
vs
✓ Proposed: Asynchronous Loops
  • ✓Humans define constraints, agents execute within them
  • ✓Agent runs continuously, unblocked
  • ✓System speed = agent speed
  • ✓Humans review patterns, not individual outputs

Core Principle: Constrain, Don't Gate

Instead of humans approving each unit of agent work, humans define the boundaries agents operate within. The two loops influence each other through asynchronous interfaces.

🧱

Platform SDKs & Templates

Agents build on sanctioned building blocks. More deterministic output, smaller error surface.

📜

Policies as Code

Security, quality, and compliance rules encoded and enforced automatically. Agents cannot violate them.

⚡

Thresholds & Circuit Breakers

Agents operate autonomously within tolerances. Breaches trigger alerts, not blocking gates.

📊

Feedback Signals, Not Approvals

Humans review aggregate output and failure patterns. Insights flow back as configuration changes.

1

Product

🔄 Human Loop
Weekly / Quarterly
  • Define product vision, strategic priorities, and quarterly themes
  • Review agent-generated backlog rankings; override where business context demands
  • Maintain and refine acceptance criteria templates and scoring weights
  • Conduct stakeholder alignment and organizational trade-off decisions
  • Review agent-drafted user stories in batches; calibrate templates if quality drifts
Async
⚡ Agent Loop
Continuous
  • Analyse user behaviour data, support tickets, and market signals continuously
  • Score and rank backlog against configurable weighted criteria
  • Draft user stories and acceptance criteria using human-maintained templates
  • Generate competitive intelligence summaries on defined cadence
  • Run A/B test analysis and surface statistically significant results
▶ Humans priority → agents
◀ Agents ranked backlog → humans
⚠ Low-confidence items → human exception
Guardrails
🛡 Agents cannot promote items to "ready" without completeness check
🛡 Scoring weight changes require human approval & version control
🛡 Agent-generated stories tagged & subject to periodic batch audit
2

Engineering

🔄 Human Loop
Weekly / Sprint
  • Build and maintain the Platform SDK — core libraries, design system, shared patterns (AI-assisted, not agentic)
  • Ensure all architectural decisions, patterns, and context live in the repo as versioned artifacts — knowledge in Slack or Confluence is invisible to agents
  • Analyse agent output quality: defect rates, test coverage, code quality indicators
  • Review production incidents; identify root causes in agent behaviour or SDK gaps
  • Tune agent configuration: prompt templates, code generation rules, thresholds
  • Process SDK change requests from agents — evaluate, prioritize, implement
Async
⚡ Agent Loop
Continuous
  • Pick up prioritized tickets from the backlog
  • Design a solution within Platform SDK constraints
  • Implement → test → check against concrete criteria → iterate until pass (the execution loop: act, verify, feed back, repeat)
  • Run SAST, linting, and code quality scans automatically; query live observability (logs, traces) to reproduce and verify runtime behaviour
  • Prepare releases and update developer/agent documentation in-repo as versioned artifacts
  • Garbage collection: background agents scan on cadence for stale docs, architectural drift, and constraint violations — open small auto-mergeable remediation PRs
  • If blocked by SDK limitation, create an SDK change request ticket
▶ The Platform SDK is the primary interface — constrains what agents can do and how
◀ Agents produce quality metrics & failure logs → humans identify systemic issues
⚠ Circuit breaker: N failures → ticket routed to human loop with full context
Guardrails
🛡 Agents cannot modify Platform SDK — only application code on top of it
🛡 All agent code must pass automated pipeline (SAST, tests, lint) to merge
🛡 Releases go through automated validation gates
🛡 Circuit breaker at configurable failure threshold
🛡 Agent commits must include structured rationale — execution plans and ADRs are compliance-auditable artifacts
🛡 Agent instruction files kept concise (<100 lines as table of contents); full context lives in structured docs/ — overloaded context degrades output quality

QA

🔄 Human Loop
Weekly / Sprint
  • Define and maintain the test strategy — testing types per risk tier, coverage thresholds, perf baselines
  • Review quality dashboards: defect escape rates, flaky test trends, coverage gaps
  • Investigate edge cases agents consistently miss; feed back as new test patterns
  • Maintain exploratory testing checklists for high-risk areas
  • Calibrate test generation templates and classification rules
Async
⚡ Agent Loop
Continuous
  • Generate test scripts with 1:1 traceability to acceptance criteria
  • Execute full regression suites, integration tests, and performance benchmarks
  • Perform fuzz testing and boundary analysis on new endpoints
  • Generate coverage reports and gap analysis against acceptance criteria
  • Auto-classify failures: code defect, test defect, or environment issue
▶ Humans define risk tiers & coverage thresholds → agents generate against them
◀ Agents produce coverage reports & failure classifications → humans calibrate templates
⚠ Systematic edge-case misses → routed to human exploratory testing
Guardrails
🛡 Agents cannot mark a story as tested without coverage threshold met
🛡 High-risk tier tests require human sign-off before release
🛡 Test classification rules (code vs test vs environment defect) version-controlled and human-approved
🛡 Every auto-closed test failure has logged rationale — auditable trace

Security

🔄 Human Loop
Weekly / On-demand
  • Define and maintain security policies — allowed, blocked, exception-required
  • Conduct architecture-level threat modelling for new systems
  • Lead incident response and forensic analysis
  • Batch-review agent triage decisions; validate false positive patterns
  • Interpret regulatory changes → update policy-as-code rules
  • Approve security exceptions (time-boxed, tracked, compensating controls)
Async
⚡ Agent Loop
Continuous
  • Run continuous SAST and dependency vulnerability scanning
  • Enforce security policies as code in CI/CD — block violating builds
  • Automated threat model analysis on new PRs (data flows, endpoints, auth)
  • Monitor runtime signals: anomalous traffic, credential exposure, config drift
  • Generate compliance evidence artifacts (audit logs, control attestations) — structured for regulatory traceability
  • Triage findings: auto-close known false positives with logged rationale
▶ Humans publish policies as code & threat model baselines → agents enforce continuously
◀ Agents surface vuln reports & triage summaries → humans review periodically
⚠ Novel vulnerability types or ambiguous risk → routed to human review
Guardrails
🛡 Agents cannot grant security exceptions — humans only
🛡 Critical/high vulns block promotion; agents cannot override
🛡 Policy changes version-controlled, require human approval
🛡 Every auto-closed finding has auditable logged rationale
🛡 Pen testing remains human-led

DevOps

🔄 Human Loop
Monthly / On-demand
  • Define infrastructure architecture patterns & templates — sanctioned deployment methods
  • Set capacity plans, cost budgets, and SLA targets
  • Design disaster recovery and business continuity strategies
  • Review incident post-mortems; identify monitoring and remediation gaps
  • Manage vendor relationships and evaluate new tooling
  • Update runbooks and remediation playbooks based on incident trends
Async
⚡ Agent Loop
Continuous
  • Monitor infrastructure health, app performance, and cost metrics
  • Execute deployments via automated pipelines (blue/green, canary) per runbooks
  • Auto-scale infrastructure within defined boundaries based on load
  • Detect & respond to incidents: correlate alerts, identify root cause, auto-remediate known patterns
  • Manage IaC — apply drift corrections, update configs per policy
  • Provision environments for new services using sanctioned templates
▶ Humans define infra templates & runbooks → agents execute within them
◀ Agents surface incident patterns & cost anomalies → humans update playbooks
⚠ Novel incident types or budget breach → escalated to human loop
Guardrails
🛡 Agents can only deploy via sanctioned pipeline templates — no ad-hoc execution
🛡 Production promotion requires automated validation gate; agents cannot bypass
🛡 Cost and capacity changes above threshold require human approval
🛡 All agent-applied IaC changes logged with rationale — full audit trail
🛡 Disaster recovery and business continuity procedures remain human-authored and human-tested

Design

The design system is the UX harness. A versioned component library with tokens, interaction patterns, and accessibility rules is the Platform SDK equivalent for the front end. Without it, agents generating UI produce inconsistent interfaces that are expensive to fix and impossible to audit. Designers maintain the constraint layer; agents build within it.
🔄 Human Loop
Weekly / Sprint
  • Define and maintain the design system — tokens, components, interaction patterns, and accessibility standards; the constraint layer for all agent-generated UI
  • Conduct user research, usability testing, and journey mapping; feed findings into Product human loop as weighted scoring inputs
  • Define information architecture and interaction principles for new product areas
  • Review agent-generated UI for brand consistency, accessibility compliance, and interaction quality — review batches, not individual outputs
  • Evaluate agent-surfaced design system gap requests; approve additions and version the system
  • Maintain component generation templates and quality thresholds; calibrate when agent output drifts
Async
⚡ Agent Loop
Continuous
  • Generate UI implementations using design system components and tokens — no ad-hoc patterns
  • Run accessibility scans (WCAG compliance) on all generated interfaces; block promotion on failures
  • Apply design tokens consistently across generated surfaces; flag deviations
  • Generate component usage documentation and design-to-implementation traceability artifacts
  • Detect design system coverage gaps — surface as design system change request tickets when a required pattern is absent
▶ Humans publish versioned design system → agents build within it
◀ Agents surface design system gap requests & accessibility reports → humans evaluate and extend
⚠ Novel interaction patterns or brand judgment → human loop only
Guardrails
🛡 Agents cannot introduce UI patterns outside the design system — must raise a change request
🛡 Accessibility failures (WCAG AA minimum) block promotion; agents cannot override
🛡 Design system versions are controlled and human-approved — agents consume, never modify
🛡 Agent-generated UI tagged and subject to periodic design review batch
🛡 User research and journey findings remain human-authored inputs — not agent-synthesised without validation

Emerging Roles

Platform SDK Engineer

Build and maintain the core libraries, patterns, and architectural building blocks that agents use. Write the "rules of the game" for agent-generated code.

← Senior/Staff Developers, Architects

Agent Loop Engineer

Configure, monitor, and optimize agent loops across all domains. Tune prompts, thresholds, scoring weights, and failure policies. Bridge human intent and agent behaviour.

← Senior Devs, DevOps, QA Leads

Agent Quality Analyst

Monitor agent output quality across all domains. Analyse defect patterns, coverage gaps, triage accuracy, and failure modes. Feed findings back as config changes.

← QA Engineers, Data Analysts, BAs

AI Policy Engineer

Translate security, compliance, and governance requirements into machine-enforceable rules (policy as code). Bridge regulatory language and automated enforcement.

← Security Eng, Compliance, Risk

Product Strategist

Focus on vision, stakeholder alignment, and strategic trade-offs. Operational product work (stories, grooming, scoring) handled by agents. Shift to higher-order judgment.

← Product Managers, Business Analysts

Design Systems Architect

Build and govern the versioned design system — the constraint layer for agent-generated UI. Define tokens, components, interaction patterns, and accessibility standards. Write the "rules of the game" for agent-generated interfaces.

← Senior UX/UI Designers, Design Leads

Incident Strategist

Focus on novel incidents, post-mortem analysis, and updating automated remediation playbooks. Routine incident response handled by agents.

← SREs, Ops Engineers, DevOps

Key Insights
  • Humans set constraints, agents execute within them — loops influence each other asynchronously
  • Agents run at agent speed; humans review patterns, not individual outputs
  • In regulated environments, agent reasoning is an audit artifact — commit rationale, ADRs, and execution plans are compliance evidence, not developer convenience
  • Knowledge not in the repo does not exist for the agent — move all governing context into versioned, repo-local artifacts
  • These loops are bets with expiration dates. Build the thinnest layer between human intent and shipped software. Treat every layer as disposable — know when to rip it out as model capability increases
  • Over time, agent loops become more capable and human loops become more strategic
